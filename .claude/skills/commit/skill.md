# Japanese Git Commit Skill

日本語でコミットメッセージを生成し、git commitを実行するスキル。

## 使い方

`/commit` と入力するだけでOK。オプションで対象ファイルや追加メッセージを指定できる。

## 動作手順

1. `git add ./` を実行してすべての変更をステージングする
2. `git diff --cached` でステージ済みの変更差分を取得する
3. 差分を分析し、変更の本質を把握する
4. 以下のルールに従って日本語コミットメッセージを起草する
5. **起草したコミットメッセージをユーザーに提示し、確認を求める**
6. ユーザーが承認したら `git commit` を実行する。修正を求められた場合はメッセージを修正してから実行する

## コミットメッセージのルール

- **1行目（タイトル）**: 50文字以内の簡潔な日本語で「何をしたか」を記述
  - 動詞で始める（追加、修正、削除、リファクタリング、更新、整備、改善など）
  - 例: `JWTリフレッシュトークン機能を追加し、Cookie管理を整備`
- **空行**
- **本文（任意）**: 変更の背景・理由・詳細を箇条書きで記述
  - `- ` で始める箇条書き
  - 「何をしたか」より「なぜしたか」「どう変わったか」を重視

## 変更がない場合

`git add ./` 後に差分がなければ、コミットするものがない旨をユーザーに伝える。

## 実行例

```
/commit
/commit src/main/java/com/example/Foo.java
```
